=== Algebraic Data Types (ADT)
[[algebraic-data-types-definition]]Алгебраїчні типи даних (Algebraic Data Types або ADT) — це поняття яке належить одразу до кількох фундаментальних сфер таких як:

* Functional Programming. ADT — це база таких мов, як Haskell, Scala, Rust, F#. В Java вони з'явилися саме як спроба запозичити кращі практики з функціонального світу. Це тема про «незмінність» (Immutability) та декларативний підхід до обробки даних;
* Type Theory. Це теоретичний фундамент. ADT тут розглядаються як математичні структури. Якщо ви чуєте слова «сильна типізація», «ієрархія типів» або «безпека типів» — ADT стоять у самому центрі цих дискусій.
* Domain Modeling. В контексті Java це тема про те, як правильно описувати бізнес-логіку. Замість складних ієрархій успадкування (класичний OOP), ADT пропонують моделювати дані як чіткі стани системи. Це тісно пов’язано з:
** Domain Driven Design (DDD) — опис сутностей через стани;
** Pattern Matching — сучасний спосіб обробки розгалужень у коді.

Термін «алгебраїчні» виник тому, що ці типи будуються за допомогою двох основних операцій, в основі ADT лежать дві ідеї: «І» та «АБО»., які нагадують додавання та множення:

1. [[algebraic-data-types-product-type]]Тип-добуток (Product Type) — це «І».
Логіка: «І» (тип A і тип B). Це структура, де значення містить усі вказані поля одночасно. Прикладом є точка на карті має координату x І координату y одночасно. В Java це звичайний `class` або `record`.

2. [[algebraic-data-types-sum-type]]Тип-сума (Sum Type) — це «АБО».
Логіка: «АБО» (або тип A, або тип B). Це коли значення може бути лише одним із чітко визначених варіантів. Приклад: Світлофор може бути Червоним, АБО Жовтим, АБО Зеленим. Він не може бути всіма трьома одночасно. В Java: Це `enum` або `sealed interface`.

Довгий час Java не мала повноцінної підтримки типів-сум «з коробки» (крім простих `enum`). Проте з виходом Java 17 (і подальшим розвитком у 21 версії), ми нарешті отримали інструменти для створення справжніх ADT. В сучасній Java ADT реалізуються через комбінацію Sealed Classes (Запечатані класи) та Records.

Якщо ви шукаєте це в офіційних оновленнях мови (JEP — Java Enhancement Proposals), дивіться теми:
