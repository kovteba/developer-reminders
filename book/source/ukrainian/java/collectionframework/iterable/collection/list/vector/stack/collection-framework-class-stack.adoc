[#java-collection-framework-stack]
====== Stack
`Stack` у Java — це *стек* (LIFO — Last In, First Out), який наслідує клас `Vector<E>` і реалізує інтерфейс `List<E>`.

* `List<E>` — поведінка списку з індексованим доступом
* LIFO-семантика — останній доданий елемент видаляється першим

.Ієрархія наслідування
[source,subs="normal"]
----
Object
  └── AbstractCollection<E>
        └── AbstractList<E>
              └── <<java-collection-framework-vector,Vector<E>>>
                    └── <<java-collection-framework-stack,Stack<E>>>
----

WARNING: `Stack` є застарілим (legacy) класом з Java 1.0. Для нового коду рекомендується використовувати `Deque<E>` (наприклад, `ArrayDeque`) замість `Stack`, оскільки `Stack` наслідує `Vector`, який є синхронізованим і повільнішим.

.Конструктор Stack()
[source,java]
----
Stack<String> stack = new Stack<>();
----

*Що робить:* створює порожній стек. Всередині — порожній масив, успадкований від `Vector`. `elementCount = 0`.

*Коли використовувати:* коли потрібна LIFO-семантика і ти свідомо працюєш зі старим кодом або API, що очікує `Stack`. У новому коді — краще `Deque`.

[source,java]
----
Stack<Integer> stack = new Stack<>();
stack.push(1);
stack.push(2);
stack.push(3);
// [1, 2, 3] — вершина стеку: 3
----

NOTE: `Stack` має лише *один конструктор* — без параметрів. Немає конструктора, що приймає `Collection`. Якщо потрібно ініціалізувати стек з колекції — доведеться додавати елементи вручну або використовувати `Deque`.

[source,java]
----
// Ініціалізація зі списку — вручну
List<String> source = List.of("a", "b", "c");
Stack<String> stack = new Stack<>();
stack.addAll(source); // успадковано від Vector
// вершина стеку: "c"
----

[#java-collection-framework-stack-class-big-o-notation]
.BigO notation для Stack:
[cols="2,1,4", options="header"]
|===
|`pop()`                |O(1)       |Повертає і видаляє елемент з вершини (кінця масиву). Прямий доступ за індексом `elementCount - 1`
|`peek()`               |O(1)       |Повертає елемент з вершини *без* видалення. Прямий доступ за індексом `elementCount - 1`, не змінює стан
|`empty()`              |O(1)       |Перевіряє `size() == 0`. `size()` — поле `elementCount` у `Vector`
|===
|`search(Object o)`     |O(n)       |Лінійний пошук від вершини до дна з `equals()`. Повертає позицію від 1 (вершина) або -1 якщо не знайдено
|===
|`get(int index)`       |O(1)       |Успадковано від `Vector` — прямий доступ до масиву за індексом (але це не Stack-семантика!)
|===
|`add(E e)`             |O(1)*      |Успадковано від `Vector`, еквівалент `push`. Амортизовано O(1)
|===
|`remove(int index)`    |O(n)       |Успадковано від `Vector` — зсуває всі елементи після `index` вліво (System.arraycopy)
|===
|`remove(Object o)`     |O(n)       |Лінійний пошук по значенню + зсув елементів
|===
|`contains(Object o)`   |O(n)       |Лінійний перебір з `equals()` порівнянням, успадковано від `Vector`
|===
|`size()`               |O(1)       |Повертає поле `elementCount` з `Vector`
|===
|`clear()`              |O(n)       |Обнуляє всі елементи масиву, щоб GC міг їх зібрати, потім `elementCount = 0`
|===
|`toArray()`            |O(n)       |Копіює внутрішній масив через `Arrays.copyOf`
|===
|`iterator()`           |O(1)       |Створює ітератор, що починає з індексу 0 (дно стеку)
|===
|===
|===

NOTE: Операції `push`, `pop`, `peek`, `empty`, `search` — це власні Stack-методи. Решта успадкована від `Vector` і порушує LIFO-інкапсуляцію: через `get(0)`, `add(0, e)`, `remove(0)` можна маніпулювати дном стеку, що є поганою практикою.

.Внутрішній устрій
`Stack` не має власних полів — весь стан зберігається у `Vector<E>`:

[source,java]
----
// У Vector:
protected Object[] elementData; // внутрішній масив
protected int elementCount;     // кількість елементів
protected int capacityIncrement; // крок росту (0 = подвоєння)
----

Стек — це просто масив, де *кінець масиву є вершиною стеку*. `push` додає в кінець, `pop` бере з кінця.

Візуалізація стеку `[1, 2, 3]` (3 — вершина):

[source]
----
індекс:    0    1    2
масив:   [ 1 ][ 2 ][ 3 ]
                      ↑
                   вершина (elementCount - 1)
----

.push(E item)
[source,java]
----
stack.push(4);
// До:    [1][2][3]
// Після: [1][2][3][4]
----

Що відбувається всередині:

[source,java]
----
public E push(E item) {
    addElement(item); // метод Vector — додає в кінець масиву
    return item;
}

// У Vector.addElement():
// якщо elementCount == elementData.length → росте масив (копіювання O(n))
// elementData[elementCount++] = obj; // інакше — просто O(1)
----

NOTE: Зростання масиву: якщо `capacityIncrement > 0` — збільшується на `capacityIncrement`, інакше — *подвоюється*. Амортизовано O(1).

.pop()
[source,java]
----
stack.pop();
// До:    [1][2][3]
// Після: [1][2]
// Повертає: 3
----

[source,java]
----
public synchronized E pop() {
    E obj;
    int len = size();
    obj = peek();                  // отримуємо вершину
    removeElementAt(len - 1);      // видаляємо останній елемент
    return obj;
}
----

.peek()
[source,java]
----
stack.peek();
// Стек: [1][2][3]
// Повертає: 3, стек не змінюється
----

[source,java]
----
public synchronized E peek() {
    int len = size();
    if (len == 0)
        throw new EmptyStackException();
    return elementAt(len - 1); // прямий доступ до масиву — O(1)
}
----

.search(Object o)
[source,java]
----
stack.push("a");
stack.push("b");
stack.push("c");
// Стек знизу вверх: [a][b][c]

stack.search("b"); // → 2  (вершина = 1, дно = size)
stack.search("a"); // → 3
stack.search("z"); // → -1
----

[source,java]
----
public synchronized int search(Object o) {
    int i = lastIndexOf(o); // пошук з кінця (вершини)
    if (i >= 0)
        return size() - i;  // перетворює індекс масиву → позицію від вершини
    return -1;
}
----

WARNING: `search` повертає *відстань від вершини*, а не індекс масиву. Вершина = 1, елемент під нею = 2 і т.д. Це легко переплутати з `indexOf`, який повертає звичайний індекс.

.Stack vs Deque — сучасна альтернатива
[source,java]
----
// Застарілий підхід:
Stack<String> stack = new Stack<>();
stack.push("x");
stack.pop();

// Сучасний підхід (рекомендовано):
Deque<String> stack = new ArrayDeque<>();
stack.push("x");   // addFirst
stack.pop();       // removeFirst
----

[cols="3,1,1", options="header"]
|===
|Доступ до дна стеку            |❌ можливий через Vector|✅ заблоковано семантикою
|Продуктивність                 |❌ нижча                |✅ вища
|Рекомендовано для нового коду  |❌                      |✅
|===
|Реалізує List                  |✅ (але це мінус)       |❌
|===
|===
|===

IMPORTANT: Документація Java явно рекомендує `Deque` замість `Stack`: _"A more complete and consistent set of LIFO stack operations is provided by the Deque interface"_.

[[java-collection-framework-stack-pitfalls]]
.Підводні камені та нюанси

.Stack IS-A Vector — порушення LIFO-семантики
[source,java]
----
Stack<String> stack = new Stack<>();
stack.push("a");
stack.push("b");
stack.push("c");
// Стек: [a, b, c] — вершина: "c"

// Успадковані Vector-методи порушують LIFO-семантику:
stack.add(0, "BOTTOM"); // вставляє в ДНО стека через Vector.add(index, e)!
stack.get(0);           // читає з дна стека — Vector.get(index)
stack.remove(0);        // видаляє з дна — Vector.remove(index)
stack.set(1, "x");      // замінює другий елемент знизу — Vector.set(index, e)
----

WARNING: `Stack` успадковує від `Vector` понад 50 публічних методів яких не повинно бути у стека (включно з `add(index, e)`, `remove(index)`, `get(index)`, `set(index, e)`, `subList()` та ін.). Це *порушує LIFO-інкапсуляцію* і є головною причиною чому `Stack` вважається поганим дизайном. Використовуй `ArrayDeque` замість `Stack`.

.Рекомендована заміна — ArrayDeque
[source,java]
----
// ЗАСТАРІЛИЙ код — Stack:
Stack<String> stack = new Stack<>();
stack.push("a");
stack.push("b");
String top = stack.pop();  // "b"
String peek = stack.peek(); // "a"
boolean empty = stack.empty(); // false

// СУЧАСНИЙ код — ArrayDeque як стек (рекомендовано):
Deque<String> stack = new ArrayDeque<>();
stack.push("a");           // addFirst()
stack.push("b");           // addFirst()
String top = stack.pop();  // removeFirst() → "b"
String peek = stack.peek(); // peekFirst() → "a"
boolean empty = stack.isEmpty(); // false
// ArrayDeque: не синхронізований, немає зайвих Vector-методів, швидший
----

.search() — 1-based індексація (не 0-based!)
[source,java]
----
Stack<String> stack = new Stack<>();
stack.push("a"); // індекс масиву: 0
stack.push("b"); // індекс масиву: 1
stack.push("c"); // індекс масиву: 2 — вершина

int pos1 = stack.search("c"); // → 1  (вершина = 1)
int pos2 = stack.search("b"); // → 2  (під вершиною = 2)
int pos3 = stack.search("a"); // → 3  (дно = 3)
int pos4 = stack.search("z"); // → -1 (не знайдено)
// "c" → 1, "b" → 2, "a" → 3 (1-based від вершини!)
----

WARNING: `search()` повертає *1-based позицію від вершини*, а не 0-based індекс масиву. Це легко переплутати з `indexOf()` (який повертає 0-based індекс від дна). Значення `1` означає вершина, `size()` означає дно.
