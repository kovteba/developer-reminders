[#java-collection-framework-hash-set]
====== HashSet class
`Hashset` - це клас який реалізує інтерфейс `Set` який у свою чергу розширяє інтерфейс `Collection`. Він представляє собою колекцію, яка не містить дублікатів елементів і використовує `HashMap` для зберігання хеш-кодів елементів. `HashSet` зберігає елементи в хеш-таблиці, яка забезпечує постійний час виконання операцій додавання, видалення та пошуку елементів. `HashSet` не гарантує порядок елементів під час ітерації. `HashSet` дозволяє зберігати тільки один нульовий елемент. `HashSet` не є синхронізованим, тому якщо необхідно забезпечити багатопоточний доступ до екземпляру `HashSet`, необхідно забезпечити зовнішню синхронізацію.

Складається із так званий баскетов (buckets) і використовує хеш-коди для зберігання елементів.

Як що два об'єкти мають однаковий хеш-код то вони зберігаються в одному баскеті. Якщо ви викликаєте `add()` метод для додавання елемента в `HashSet`, то він викликає `hashCode()` метод для обчислення хеш-коду елемента і вставляє його в баскет. Якщо в баскеті вже є елементи, то він викликає `equals()` метод для перевірки наявності дублікатів. Якщо `equals()` повертає `false`, тоді елемент додається в баскет. Якщо `equals()` повертає `true`, тоді елемент не додається в баскет.

Пошук елемента в `HashSet` займає постійний час, оскільки він використовує хеш-коди для зберігання елементів. Якщо два об'єкти мають однаковий хеш-код, то вони зберігаються в одному баскеті. Якщо ви викликаєте `contains()` метод для пошуку елемента в `HashSet`, то він викликає `hashCode()` метод для обчислення хеш-коду елемента і виконує пошук в баскеті. Якщо в баскеті вже є елементи, то він викликає `equals()` метод для перевірки наявності дублікатів. Якщо `equals()` повертає `true`, тоді метод `contains()` повертає `true`, інакше він повертає `false`.

[#java-collection-framework-hash-set-class-constructor]
.Конструктори:
* `HashSet()`: создает пустой список;
* `HashSet(Collection<? extends E> col)`: создает хеш-таблицу, в которую добавляет все элементы коллекции `col`;
* `HashSet(int capacity)`: параметр `capacity` указывает начальную емкость таблицы, которая по умолчанию равна `16`;
* `HashSet(int capacity, float koef)`: параметр `koef` или коэффициент заполнения, значение которого должно быть в пределах от 0.0 до 1.0, указывает, насколько должна быть заполнена емкость объектами прежде чем произойдет ее расширение. Например, коэффициент 0.75 указывает, что при заполнении емкости на 3/4 произойдет ее расширение.

[#java-collection-framework-hash-set-class-methods]
.Методи:
* `add(E e)`: добавляет элемент в хеш-таблицу;
* `remove(Object o)`: удаляет элемент из хеш-таблицы;
* `contains(Object o)`: проверяет наличие элемента в хеш-таблице;
* `size()`: возвращает количество элементов в хеш-таблице;
* `isEmpty()`: проверяет пуста ли хеш-таблица;
* `clear()`: удаляет все элементы из хеш-таблицы;
* `iterator()`: возвращает итератор для хеш-таблицы;
* `spliterator()`: возвращает сплитератор для хеш-таблицы;
* `forEach(Consumer<? super E> action)`: выполняет действие для каждого элемента в хеш-таблице;
* `removeIf(Predicate<? super E> filter)`: удаляет все элементы, которые соответствуют предикату;
* `toArray()`: возвращает массив, содержащий все элементы хеш-таблицы;
* `toArray(T[] a)`: возвращает массив, содержащий все элементы хеш-таблицы;
* `addAll(Collection<? extends E> c)`: добавляет все элементы коллекции `c` в хеш-таблицу;
* `containsAll(Collection<?> c)`: проверяет наличие всех элементов коллекции `c` в хеш-таблице;
* `removeAll(Collection<?> c)`: удаляет все элементы коллекции `c` из хеш-таблицы;
* `retainAll(Collection<?> c)`: удаляет все элементы, которые не содержатся в коллекции `c`;
* `equals(Object o)`: сравнивает хеш-таблицу с объектом `o`;
* `hashCode()`: возвращает хеш-код хеш-таблицы;
* `spliterator()`: возвращает сплитератор для хеш-таблицы.

[#java-collection-framework-hash-set-class-big-o-notation]
.BigO notation для методов HashSet:
* `add(E e)`: O(1);
* `remove(Object o)`: O(1);
* `contains(Object o)`: O(1);
* `size()`: O(1);
* `isEmpty()`: O(1);
* `clear()`: O(1);
* `iterator()`: O(1);
* `spliterator()`: O(1);
* `forEach(Consumer<? super E> action)`: O(n);
* `removeIf(Predicate<? super E> filter)`: O(n);
* `toArray()`: O(n);
* `toArray(T[] a)`: O(n);
* `addAll(Collection<? extends E> c)`: O(n);
* `containsAll(Collection<?> c)`: O(n);
* `removeAll(Collection<?> c)`: O(n);
* `retainAll(Collection<?> c)`: O(n);
* `equals(Object o)`: O(n);
* `hashCode()`: O(n);
* `spliterator()`: O(n).


[[java-collection-framework-hash-set-access-by-key]] Ні, в HashSet немає можливості отримати доступ до елементу по ключу або індексу. Якщо треба отримати елемент елемент, тоді треба викликати `iterator()` або використовуйте `for()` (той же самий Iterator). Якщо вам потрібно отримувати дані по ключу, то HashSet не підходить, по-перше, вона не призначена для цього, а по-друге, складність часу пошуку елемента займає O(n).
