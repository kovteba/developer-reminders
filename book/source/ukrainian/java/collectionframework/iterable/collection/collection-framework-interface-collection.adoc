==== Collection
Інтерфейс Collection розширює інтерфейс `Iterable` і має один метод `iterator()`, який повертає об'єкт ітератора, який дозволяє послідовно переглядати всі елементи колекції.

// [source,subs="normal"]
// ----
[I] <<java-collection-framework-collection,Collection<E>>>
├── [I] <<java-collection-framework-list,List<E>>>
├── [I] <<java-collection-framework-set,Set<E>>>
└── [I] Queue<E>
// ----

Це інтерфейс є у складі `JDK 1.2` і визначає основні методи роботи з простими наборами елементів, які будуть спільними для всіх його реалізацій. Нижче наведемо деякі з найбільш часто використовуваних методів інтерфейсу `Collection`:

- `boolean add(E e)` — додає в колекцію об'єкт item. При вдалому додаванні повертає true, при невдалому - false;
- `boolean addAll(Collection<? extends E> c)` — додає в колекцію всі елементи з колекції col. При вдалому додаванні повертає true, при невдалому - false;
- `void clear()` — видаляє всі елементи з колекції;
- `boolean contains(Object o)`— повертає true, якщо об'єкт item міститься в колекції, інакше повертає false;
- `boolean isEmpty()` — повертає true, якщо колекція порожня, інакше повертає false;
- `Iterator<E> iterator()` — повертає об'єкт Iterator для обходу елементів колекції;
- `boolean remove(Object o)` — повертає true, якщо об'єкт item успішно видалений з колекції, інакше повертає false;
- `boolean removeAll(Collection<?> c)` — видаляє всі об'єкти колекції col з поточної колекції. Якщо поточна колекція змінилася після видалення, повертає true, інакше повертає false;
- `boolean retainAll(Collection<?> c)` — видаляє всі об'єкти з поточної колекції, крім тих, які містяться в колекції col. Якщо поточна колекція змінилася після видалення, повертає true, інакше повертає false;
- `int size()` — повертає кількість елементів в колекції;
- `Object[] toArray()` — повертає масив, який містить всі елементи колекції.

Інтерфейс був трохи змінений з приходом дженериків в `Java 1.5`. Також в версії `Java 8` було додано кілька нових методів для роботи з лямбдами:

- `default boolean removeIf(Predicate<? super T> filter)` — видаляє всі елементи колекції, які задовольняють умову filter;
- `default Stream<E> stream()` — повертає об'єкт Stream для обходу елементів колекції;
- `default Stream<E> parallelStream()` — повертає паралельний об'єкт Stream для обходу елементів колекції.

ArrayList:

// * доступ к произвольному элементу по индексу за константное время O(1);
// * доступ к элементам по значению за линейное время O(N);
// * вставка в конец в среднем производится за константное время O(1);
// * удаление произвольного элемента из списка занимает значительное время т.к. при этом все элементы, находящиеся «правее» смещаются на одну ячейку влево (реальный размер массива (capacity) не изменяется);
// * вставка элемента в произвольное место списка занимает значительное время т.к. при этом все элементы, находящиеся «правее» смещаются на одну ячейку вправо;
* минимум накладных расходов при хранении.

LinkedList:

// * на получение элемента по значению потребуется линейное время O(N);
// * на получение элемента по индексу  потребуется линейное время O(N);
// * но доступ к первому и последнему элементу списка всегда осуществляется за константное время O(1) — ссылки постоянно хранятся на первый и последний элемент;
// * на добавление и удаление в начало или конец списка потребуется константное O(1);
// * вставка или удаление в/из произвольного место константное O(1);
* но поиск позиции вставки и удаления за линейное время O(N);
* требует больше памяти для хранения такого же количества элементов, потому что кроме самого элемента хранятся еще указатели на следующий и предыдущий элементы списка.







 Реалізації:
** `ArrayList` інкапсулює в собі звичайний масив, довжина якого автоматично збільшується при додаванні нових елементів.
** `LinkedList` (двунаправлений зв'язний список) - складається з вузлів, кожен з яких містить як саме дані, так і дві посилання на наступний і попередній вузол.
** `Vector` реалізація динамічного масиву об'єктів, методи якого синхронізовані.
** `Stack` реалізація стека LIFO (last-in-first-out).
* `Set` описує неупорядковану колекцію, яка не містить дублюючі елементи. Реалізації:
** `HashSet` використовує HashMap для зберігання даних. В якості ключа і значення використовується додаваний елемент. Послідовність елементів не гарантується при додаванні.
** `LinkedHashSet` гарантує, що порядок елементів при обході колекції буде ідентичний порядку додавання елементів.
** `TreeSet` надає можливість керувати порядком елементів в колекції за допомогою об'єкта Comparator, або зберігає елементи з використанням «natural ordering».
* `Queue` призначена для зберігання елементів з попередньо визначеним способом вставки і вилучення FIFO (first-in-first-out):
** `PriorityQueue` надає можливість керувати порядком елементів в колекції за допомогою об'єкта Comparator, або зберігає елементи з використанням «natural ordering».
** `ArrayDeque` реалізація інтерфейса Deque, який розширює інтерфейс Queue методами, які дозволяють реалізувати конструкцію виду LIFO (last-in-first-out).
