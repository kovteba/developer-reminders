[#java-collection-framework-sortedmap]
=== SortedMap interface
`SortedMap<K,V>` — це інтерфейс, який розширює `Map<K,V>` і гарантує, що всі ключі *відсортовані*. Сортування відбувається або за *натуральним порядком* (якщо ключі реалізують `Comparable`), або за порядком заданого `Comparator`.

// [source,subs="normal"]
// ----
[I] <<java-collection-framework-map,Map<K,V>>
└── [I] <<java-collection-framework-sortedmap,SortedMap<K,V>>>
    └── [I] <<java-collection-framework-navigablemap,NavigableMap<K,V>>>
        ├── [C] <<java-collection-framework-treemap,TreeMap<K,V>>>
        └── [CC] ConcurrentSkipListMap<K,V>
// ----

[[java-collection-framework-sortedmap-methods]]
.SortedMap — методи та ієрархія інтерфейсів
`SortedMap<K,V>` додає до `Map`: `comparator()` — повертає Comparator або null (якщо натуральний порядок); `firstKey()`, `lastKey()` — O(log n); `headMap(toKey)` — ключі < toKey; `tailMap(fromKey)` — ключі ≥ fromKey; `subMap(fromKey, toKey)` — ключі [from, to). `comparator()` повертає null коли використовується натуральний порядок (Comparable) — це нормально. Ієрархія: `Map` → `SortedMap` → `NavigableMap` → `ConcurrentNavigableMap`. Реалізації: `TreeMap` (NavigableMap), `ConcurrentSkipListMap` (ConcurrentNavigableMap).

[#java-collection-framework-sortedmap-interface-methods]
.Методи інтерфейсу `SortedMap`:
* `Comparator<? super K> comparator()` — повертає `Comparator` що використовується для впорядкування, або `null` якщо використовується натуральний порядок;
* `K firstKey()` — повертає перший (найменший) ключ в мапі. Кидає `NoSuchElementException` якщо мапа порожня;
* `K lastKey()` — повертає останній (найбільший) ключ в мапі. Кидає `NoSuchElementException` якщо мапа порожня;
* `SortedMap<K,V> headMap(K toKey)` — повертає view мапи з усіма ключами *строго менше* `toKey`. View є live — зміни відображаються в оригінальній мапі;
* `SortedMap<K,V> tailMap(K fromKey)` — повертає view мапи з усіма ключами *більше або рівними* `fromKey`;
* `SortedMap<K,V> subMap(K fromKey, K toKey)` — повертає view мапи з ключами від `fromKey` (включно) до `toKey` (не включно).

NOTE: Всі три методи повертають *live view* — зміни в повернутій під-мапі відображаються в оригінальній і навпаки. Спроба додати елемент поза межами діапазону кидає `IllegalArgumentException`.

[source,java]
----
TreeMap<Integer, String> map = new TreeMap<>();
map.put(1, "one");
map.put(3, "three");
map.put(5, "five");
map.put(7, "seven");

SortedMap<Integer, String> head = map.headMap(5);   // {1=one, 3=three}
SortedMap<Integer, String> tail = map.tailMap(5);   // {5=five, 7=seven}
SortedMap<Integer, String> sub  = map.subMap(3, 7); // {3=three, 5=five}

System.out.println(map.firstKey()); // 1
System.out.println(map.lastKey());  // 7
----
