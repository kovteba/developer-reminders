[#java-collection-framework-navigablemap]
=== NavigableMap interface
`NavigableMap<K,V>` — це інтерфейс, який розширює `<<java-collection-framework-sortedmap,SortedMap<K,V>>>` і надає методи для *навігації по ключах*: знаходження найближчого ключа знизу/зверху, ітерації у зворотному порядку, діапазонних запитів з гнучким вибором включення меж.

[[java-collection-framework-navigablemap-methods]]
.NavigableMap — навігаційні методи та відмінності від SortedMap
`NavigableMap` додає до `SortedMap`: floor/ceiling/lower/higher (Entry та Key варіанти); `pollFirstEntry()` та `pollLastEntry()` (видалення + повернення, відмінність від firstEntry/lastEntry що не видаляють); `descendingMap()` — O(1) reverse view; `navigableKeySet()`, `descendingKeySet()`. `headMap(toKey, boolean inclusive)` — NavigableMap версія дозволяє включати toKey (SortedMap.headMap має тільки exclusive). `subMap(fromKey, fromInclusive, toKey, toInclusive)` — повний контроль над межами. `ConcurrentNavigableMap` = `ConcurrentMap` + `NavigableMap`, реалізований `ConcurrentSkipListMap`.

[#java-collection-framework-navigablemap-interface-methods]
.Методи інтерфейсу `NavigableMap`:

.Методи пошуку найближчого ключа
* `K ceilingKey(K key)` — повертає найменший ключ `k ≥ key`, або `null` якщо такого немає;
* `Map.Entry<K,V> ceilingEntry(K key)` — як `ceilingKey`, але повертає пару ключ-значення;
* `K floorKey(K key)` — повертає найбільший ключ `k ≤ key`, або `null`;
* `Map.Entry<K,V> floorEntry(K key)` — як `floorKey`, але повертає пару;
* `K higherKey(K key)` — повертає найменший ключ `k > key` (строго більше), або `null`;
* `Map.Entry<K,V> higherEntry(K key)` — як `higherKey`, але повертає пару;
* `K lowerKey(K key)` — повертає найбільший ключ `k < key` (строго менше), або `null`;
* `Map.Entry<K,V> lowerEntry(K key)` — як `lowerKey`, але повертає пару.

.Методи першого/останнього елемента
* `Map.Entry<K,V> firstEntry()` — повертає пару з найменшим ключем, або `null` якщо порожня;
* `Map.Entry<K,V> lastEntry()` — повертає пару з найбільшим ключем, або `null`;
* `Map.Entry<K,V> pollFirstEntry()` — повертає *і видаляє* пару з найменшим ключем;
* `Map.Entry<K,V> pollLastEntry()` — повертає *і видаляє* пару з найбільшим ключем.

.Методи зворотного порядку та ключів
* `NavigableMap<K,V> descendingMap()` — повертає view мапи у зворотному порядку ключів;
* `NavigableSet<K> navigableKeySet()` — повертає `NavigableSet` всіх ключів у зростаючому порядку;
* `NavigableSet<K> descendingKeySet()` — повертає `NavigableSet` всіх ключів у спадаючому порядку.

.Діапазонні методи з гнучкими межами
* `NavigableMap<K,V> headMap(K toKey, boolean inclusive)` — view від початку до `toKey`; `inclusive` визначає чи включати `toKey`;
* `NavigableMap<K,V> tailMap(K fromKey, boolean inclusive)` — view від `fromKey` до кінця; `inclusive` визначає чи включати `fromKey`;
* `NavigableMap<K,V> subMap(K fromKey, boolean fromInclusive, K toKey, boolean toInclusive)` — view між `fromKey` і `toKey` з контролем включення обох меж.

[source,java]
----
TreeMap<Integer, String> map = new TreeMap<>();
map.put(1, "one"); map.put(3, "three"); map.put(5, "five");
map.put(7, "seven"); map.put(9, "nine");

map.ceilingKey(4);  // → 5  (найменший ≥ 4)
map.floorKey(4);    // → 3  (найбільший ≤ 4)
map.higherKey(5);   // → 7  (строго > 5)
map.lowerKey(5);    // → 3  (строго < 5)

map.firstEntry();   // → 1=one
map.pollLastEntry();// → 9=nine (і видаляє!)

// Зворотний порядок:
map.descendingMap().forEach((k, v) ->
    System.out.println(k + "=" + v)
); // 7, 5, 3, 1

// Діапазон [3, 7):
map.subMap(3, true, 7, false); // {3=three, 5=five}
// Діапазон (3, 7]:
map.subMap(3, false, 7, true); // {5=five, 7=seven}
----
